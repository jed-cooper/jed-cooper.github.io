DATA ANALYSIS
PROJECT 3

SUMMARY

This data project outlines the locations of landslide occurences in India from 2007 to 2016; it's primary aim was the creation of an interactive map visualising all recorded landslides for users so they can explore the data for themselves.

Accessing an event will display it's date, location, fatality/injury count, trigger and information source, with the size of each marker being proportional to the severity of the incident.

The prototype was designed using geopandas and folium, with visualisations dependent data from NASA's Global Landslide Catalog and shapefiles from Indian Remote Sensing. Please follow this link to view the visualisation.

CHALLENGES

Visualising rainfall time-series data from satellite readings:
- plotting rainfall time-series data against landslide events was useful in assessing rainfall activity before/during each event.
- the plot generated from the data was convoluted and unprofessional, due to the dataframe containing rainfall measurements taken for multiple latitude/longitude locations on the same date.
- to fix this, a new dataset could be produced that approximates rainfall measurements from satellite readings with those of local weather stations situated closest to event locations.

Overlaying a time-series of weather conditions using folium:
- mapping rainfall data to landslide events gives the visualisation more utility, allowing people to see the dispersion and intensity of rainfall over areas with a high risk of landslides.
- the rainfall observations acquired from satellite data were measured at 0.25 x 0.25 degree intervals, and when plotted the visualised data became a uniforn distribution inferring only approximate rainfall levels.
- this could be fixed by verifying rainfall levels with local weather stations, or by averaging rainfall data from the four grid coordinates surrounding each landslide event.

TECHNIQUES

geopandas:
- create geometry using geodetic coordinate system (gpd.GeoDataFrame(df, geometry=gpd.points_from_xy(lat, lon), crs="EPSG:4326"))
- plot shapefile (gdf.plot(ax=map.plot()))

matplotlib:
- plot vertical lines for events (for date in df['date']: plt.axvline(date))
- reformat y-axis labels (yticks(np.arange(0, 600, step=50)))

folium:
- apply severity measure to radius (severity = (fatality+injury) / (fatality.sum()+injury.sum()) * 100)
- format popups using html (popup=folium.Popup(html))

EXTENSIONS

The project could be extended to include:
- time-series visualisations of local weather patterns.
- a larger dataset that incorporates up-to-date landslide events.
- marker shapes/colours allocated to each type of landslide trigger.
